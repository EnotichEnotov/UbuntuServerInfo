# Настройка Ubuntu мини‑ПК как SSH‑сервера (Windows CMD клиент)

## Шаг 1 — Ubuntu: Wi‑Fi + автоподключение

```bash
sudo systemctl enable --now NetworkManager
nmcli radio wifi on
nmcli dev wifi list
sudo nmcli dev wifi connect "SSID_СЕТИ(название)" password "ПАРОЛЬ_ОТ_WIFI"
sudo nmcli con modify "SSID_СЕТИ" connection.autoconnect yes connection.autoconnect-priority 100
sudo nmcli con up "SSID_СЕТИ"
```

---

## Шаг 2 — Ubuntu: SSH сервер + автозапуск

```bash
sudo apt update
sudo apt install -y openssh-server
sudo systemctl enable --now ssh
systemctl status ssh --no-pager
```

---

## Шаг 3 — Ubuntu: оставить вход по паролю (временно)

```bash
printf "PubkeyAuthentication yes\nPasswordAuthentication yes\nKbdInteractiveAuthentication yes\n" | sudo tee /etc/ssh/sshd_config.d/00-auth.conf >/dev/null
sudo sshd -t && sudo systemctl restart ssh
whoami
hostname -I
```

> Запомни:
> - `username` из `whoami`
> - `ip` из `hostname -I`

---

## Шаг 4 — Windows CMD: проверить вход по паролю

```cmd
ssh -p 22 username@ip
```

---

## Шаг 5 — Windows CMD: создать SSH‑ключ

```cmd
ssh-keygen -t ed25519 -a 100 -f "C:\Users\User\.ssh\mini_pc_ed25519" -C "mini-pc"
```

---

## Шаг 6 — Windows CMD: отправить Публичный ключ на Ubuntu

```cmd
type "C:\Users\User\.ssh\mini_pc_ed25519.pub" | ssh -p 22 username@ip "umask 077; mkdir -p ~/.ssh; touch ~/.ssh/authorized_keys; cat >> ~/.ssh/authorized_keys; chmod 700 ~/.ssh; chmod 600 ~/.ssh/authorized_keys"
```

---

## Шаг 7 — Windows CMD: вход по Приватному ключу

```cmd
ssh -i "C:\Users\User\.ssh\mini_pc_ed25519" -p 22 username@ip
```

---

## Шаг 8 — Ubuntu: отключить пароль (Для безопасности, только после успешного Шага 7)

```bash
printf "PubkeyAuthentication yes\nPasswordAuthentication no\nKbdInteractiveAuthentication no\n" | sudo tee /etc/ssh/sshd_config.d/01-key-only.conf >/dev/null
sudo sshd -t && sudo systemctl restart ssh
```

---

## Шаг 9 — Проверка автозапуска после перезагрузки(на случай аварийного выключения сервера)

Ubuntu:

```bash
sudo reboot
```

Windows CMD после загрузки Ubuntu:

```cmd
ssh -i "C:\Users\User\.ssh\mini_pc_ed25519" -p 22 username@ip
```














# Firebird на Ubuntu: установка, права, проверка, автозапуск

## Шаг 1 — установить Firebird и утилиты

```bash
sudo apt update
sudo apt install -y firebird3.0-server firebird3.0-utils
```

Во время установки:
- задайте пароль для `SYSDBA` (запомнить)
- пароль можно увидеть `sudo cat /etc/firebird/3.0/SYSDBA.password`

---

## Шаг 2 — включить автозапуск и запустить сейчас

```bash
sudo systemctl enable --now firebird3.0
systemctl status firebird3.0 --no-pager
systemctl is-enabled firebird3.0
```

Ожидаемо:
- `active (running)`
- `enabled`

---

## Шаг 3 — подготовить каталог данных

```bash
sudo mkdir -p /var/lib/firebird/3.0/data
sudo chown firebird:firebird /var/lib/firebird/3.0/data
sudo chmod 750 /var/lib/firebird/3.0/data
```

---

## Шаг 4 — исправить права временного каталога (чтобы не было Permission denied)

```bash
sudo systemctl stop firebird3.0
sudo rm -rf /tmp/firebird
sudo mkdir -p /tmp/firebird
sudo chown root:root /tmp/firebird
sudo chmod 1777 /tmp/firebird
sudo systemctl start firebird3.0
```

---

## Шаг 5 — создать тестовую БД и таблицу

```bash
sudo -u firebird isql-fb -user sysdba -password "ВАШ_ПАРОЛЬ" -q <<'SQL'
CREATE DATABASE '/var/lib/firebird/3.0/data/test.fdb';

CREATE TABLE t_ping(
  id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  msg VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

INSERT INTO t_ping(msg) VALUES ('hello firebird');
INSERT INTO t_ping(msg) VALUES ('msg 2');
INSERT INTO t_ping(msg) VALUES ('msg 3');
INSERT INTO t_ping(msg) VALUES ('msg 4');
INSERT INTO t_ping(msg) VALUES ('msg 5');
COMMIT;

SELECT COUNT(*) AS cnt FROM t_ping;
SELECT id, msg, created_at FROM t_ping ORDER BY id;

QUIT;
SQL
```

---

## Шаг 6 — дать клиентскому пользователю права для работы без sudo

Заменить `USER_LOGIN` на логин пользователя Ubuntu:

```
sudo usermod -aG firebird USER_LOGIN
```

После этого клиенту нужно перелогиниться (выйти/зайти по SSH), чтобы группа применилась (иногда применяется сразу)

Проверка: `getent group firebird`  (должно быть firebird:x:127:USER_LOGIN) ИЛИ ввести `id`, в списке групп должна быть `firebird`

---

## Шаг 7 — рабочая команда для запросов (без sudo, протестировать работу БД)

```
isql-fb -user sysdba -password "ТВОЙ_ПАРОЛЬ" /var/lib/firebird/3.0/data/test.fdb -q <<'SQL'
SELECT CURRENT_TIMESTAMP AS now_ts FROM RDB$DATABASE;
SELECT COUNT(*) AS cnt FROM t_ping;
SELECT id, msg, created_at FROM t_ping ORDER BY id;
QUIT;
SQL
```

---

## Шаг 8 — проверка автоподъёма после перезагрузки

```
sudo reboot
```

После загрузки:
```
systemctl status firebird3.0 --no-pager
```

Должно быть `active (running)`.

---

## Шаг 9 — команды диагностики

```bash
systemctl status firebird3.0 --no-pager
sudo journalctl -u firebird3.0 -n 200 --no-pager
ls -ld /tmp /tmp/firebird
ls -l /var/lib/firebird/3.0/data
```
